\chapter {Otros módulos}

\section{Introducción}
A continuación explicamos el funcionamiento del resto de los módulos. Los aglutinamos en esta sección, por carecer de interés la explicación detallada de los mismos, debido a su sencillez.

\section {Módulo de post-gestión}
Hemos añadido un módulo que recoja todas las señales que son generadas por las distintas ramas de proceso del \emph{pipeline}, y las agrupe en una sola salida válida, para ofrecer de este modo, a través de sus puertos, datos a todos los módulos de salida.

\subsection{Detalles}
\begin{itemize}
  \item {\bf Entrada}: Cadenas que representan órdenes o parámetros. Como argumentos, las cadenas por defecto.
  \item {\bf Salida}: Una estructura de tipo \emph{orden/parámetro} para el robot o el entorno 3D.
  \item {\bf Descripción}: Módulo que se encarga de recoger todo el proceso del pipeline y unificarlo en una sola estructura.
\end{itemize}


\section{Ventana de parámetros}

La ventana de parámetros es una interfaz gráfica en la que es posible elegir un color, y tres tolerancias (una para el rojo, otra para el verde y otra para el azul), y alimentar así a un módulo de filtro, de tal modo que la parametrización de los valores del mismo se pueda hacer en tiempo real.
\subsection{Detalles}
\begin{itemize}
  \item {\bf Entrada}: Sólo los argumentos del XML, con colores por defecto.
  \item {\bf Salida}: Una estructura que define colores y tolerancias sobre los mismos, para los filtros.
  \item {\bf Descripción}: Módulo que se encarga de controlar los colores del filtrado.
\end{itemize}


\section{Ventana de imágenes}
La ventana de imágenes es simplemente un módulo que muestra gráficamente, en una ventana que se redimensiona automáticamente, una imagen que sigue el formato propio de nuestra aplicación (ver sección \ref{formato_imagenes}).

La ventana de imágenes tiene como función añadida la capacidad de guardar una toma de la imagen que se este dibujando en el instante en el que se pulsa \textbf{F5}.
\subsection{Detalles}
\begin{itemize}
  \item {\bf Entrada}: Una imagen en el formato del pipeline, y la información del archivo que se guardará.
  \item {\bf Salida}: No tiene salida, más que la gráfica, por pantalla.
  \item {\bf Descripción}: Módulo que se encarga de mostrar imágenes.
\end{itemize}


\section{Módulo de control de joystick}
Hemos añadido también al pipeline un módulo que, a través de un interfaz de joystick, puede traducir sus entradas a las correspondientes órdenes y parámetros del robot y del entorno 3D. De esta forma, la depuración es sencilla y rápida.

Para la implementación hemos utilizado las bibliotecas SDL (Simple Directmedia Library).
\subsection{Detalles}
\begin{itemize}
  \item {\bf Entrada}: La entrada del joystick, y, como argumentos, la configuración de este dispositivo.
  \item {\bf Salida}: Órdenes directas para el robot.
  \item {\bf Descripción}: Módulo que traduce el control de un joystick estándar a movimientos del robot o el entorno 3D.
\end{itemize}


\section{Módulo de salida}
Este módulo es simplemente una ventana con un cuadro de texto que es capaz de imprimir en él todas las cadenas (en formato C, \verb|char *|) que le llegan a través de sus puertos.
\subsection{Detalles}
\begin{itemize}
  \item {\bf Entrada}: Cualquier texto.
  \item {\bf Salida}: El texto en una ventana.
  \item {\bf Descripción}: Módulo auxiliar que muestra texto por pantalla.
\end{itemize}
